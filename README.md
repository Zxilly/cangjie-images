# cangjie-images

`cangjie-images` is the automation project behind the community-maintained
Docker image [`zxilly/cangjie`](https://hub.docker.com/r/zxilly/cangjie).
It fetches upstream Cangjie release metadata, renders per-architecture
Dockerfiles, and prepares the build/publish matrix used by GitHub Actions.

## Scope

- Stable channels: `lts`, `sts`
- Optional dynamic channel: `nightly`
- Supported architectures: `amd64`, `arm64`
- Default image name: `zxilly/cangjie`

The repository is not the SDK itself. It contains the generation, planning,
and publishing logic for the Docker images that package the upstream SDK.

## Repository Layout

- `src/cangjie_images/`
  CLI, manifest handling, Dockerfile templating, and publish planning.
- `versions/<channel>/<version>/<base>/<arch>/Dockerfile`
  Committed stable Dockerfiles generated by `cangjie-images generate`.
- `.build-context/<slug>/Dockerfile`
  Ephemeral nightly Dockerfiles rendered during `cangjie-images plan`.
- `.github/workflows/`
  CI, version-refresh, and publish automation.

## Supported Base Variants

Stable releases are generated for these base names:

- `bookworm` (default Debian variant, based on `buildpack-deps:bookworm`)
- `slim-bookworm`
- `bullseye`
- `slim-bullseye`
- `trixie`
- `slim-trixie`
- `openeuler-24.03`
- `slim-openeuler-24.03`
- `openeuler-22.03`
- `slim-openeuler-22.03`
- `openeuler-20.03`
- `slim-openeuler-20.03`

The default base is `bookworm`. All other variants publish tags with the full
base name suffix.

## Tag Rules

Tag generation is derived from the committed Dockerfiles plus the current stable
heads seen in `versions/`.

### Stable tags

For the default `bookworm` base, raw tags and explicit `-bookworm` aliases are
both published. Example for an LTS head release:

- `1.0.5`
- `1.0.5-bookworm`
- `1.0`
- `1.0-bookworm`
- `lts`
- `lts-bookworm`
- `latest`
- `latest-bookworm`

For non-default bases, the tag always includes the exact base name:

- `1.0.5-slim-bookworm`
- `lts-slim-bookworm`
- `1.0.5-openeuler-24.03`
- `1.0.5-slim-openeuler-24.03`

### Nightly tags

Nightly Dockerfiles are rendered on demand into `.build-context/`. The default
base publishes `nightly-<version>` and `nightly`, while other bases use the
same `-<base>` suffix convention, such as:

- `nightly-<version>-slim-bookworm`
- `nightly-slim-bookworm`

## Local Usage

Install dependencies:

```console
uv sync --locked --dev
```

Generate committed stable Dockerfiles into `versions/`:

```console
uv run cangjie-images generate --versions-root versions
```

Build the publish plan for Docker Hub:

```console
uv run cangjie-images plan --image zxilly/cangjie
```

Write digest metadata for a per-architecture build job:

```console
uv run cangjie-images write-digest \
  --output-dir digests \
  --release-id lts-1-0-5-bookworm \
  --arch amd64 \
  --digest sha256:...
```

Merge uploaded digests into multi-arch tags:

```console
uv run cangjie-images merge \
  --image zxilly/cangjie \
  --release-id lts-1-0-5-bookworm \
  --tags-json '["1.0.5","1.0.5-bookworm"]' \
  --arches-json '["amd64","arm64"]' \
  --digests-dir digests
```

## Using the Published Image

The default tag still maps to the `bookworm` base. For example:

```console
docker run -it --rm zxilly/cangjie:1.0 bash
```

Non-default variants use the full base name:

```console
docker run -it --rm zxilly/cangjie:1.0-slim-bookworm cjpm build
docker run -it --rm zxilly/cangjie:1.0-openeuler-24.03 bash
```

## GitHub Workflows

- `ci.yml`
  Runs ruff, pyright, and pytest.
- `update-versions.yml`
  Regenerates committed stable Dockerfiles under `versions/` and opens a PR.
- `publish.yml`
  Compares committed Dockerfiles with Docker Hub tags, builds missing variants,
  and publishes multi-architecture manifests.
